# Security Policy

Do not report security vulnerabilities through public GitHub issues.

If you believe you have found a vulnerability in the L-GEVITY API, send a
private report with:

- the affected endpoint or integration flow
- reproduction steps
- expected impact
- relevant request IDs, without secrets or personal data

API keys are secrets. Do not include real API keys, bearer tokens, connection
strings, customer data, or private health data in issues, pull requests, logs,
screenshots, or examples.

If an API key is exposed, revoke it immediately from the L-GEVITY profile page
and create a new key.
